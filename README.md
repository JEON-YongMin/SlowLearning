# 느린 학습자를 위한 학습 지원 앱

## 프로젝트 소개

느린 학습자 및 경계선 지능 사용자의 학습을 지원하기 위한 애플리케이션의 백엔드 프로젝트입니다.  
사용자가 단어 학습, 글 요약 학습, 커뮤니티 기능을 이용할 수 있도록 REST API와 데이터 저장 구조를 구현했습니다.

본 프로젝트에서는 단순 기능 구현뿐 아니라, 사용자별 학습 기록과 요약 데이터를 안정적으로 저장하고 조회할 수 있도록 데이터 흐름과 API 구조를 고려하며 개발했습니다.

## 주요 기능

### 1. 단어 학습

- 하루 단어 퀴즈 제공
- 사용자별 단어 학습 기록 저장
- 학습 결과를 저장할 수 있는 데이터 구조 구현

### 2. 글 요약 학습

- 난이도별 글 목록 제공
- 사용자가 작성한 요약 내용 저장
- 요약 학습 데이터를 사용자별로 관리할 수 있는 구조 구현

### 3. 커뮤니티

- 게시글 작성, 조회, 수정, 삭제 기능 구현
- 댓글 작성, 조회, 수정, 삭제 기능 구현
- 게시글과 댓글의 관계를 고려한 데이터 처리 구조 구현

## 개발 과정에서 중점적으로 고려한 부분

### 데이터 흐름에 맞춘 백엔드 구조 설계

단어 학습, 글 요약 학습, 커뮤니티 기능은 각각 다른 데이터 흐름을 가지고 있었습니다.  
따라서 기능별로 필요한 데이터가 무엇인지 먼저 정리하고, 사용자와 학습 기록, 게시글, 댓글 간의 관계를 고려해 엔티티 구조와 API 흐름을 설계했습니다.

### 프론트엔드 연동을 고려한 API 설계

프론트엔드와 연동하는 과정에서 요청값과 응답값의 기준이 명확해야 한다고 판단했습니다.  
기능별 요청 파라미터, 응답 예시, 오류 상황을 정리하며 API 명세를 관리했고, 이를 바탕으로 팀원들과 개발 기준을 맞췄습니다.

## 트러블슈팅 경험

### 문제 상황

백엔드 API 구현 후 프론트엔드와 연동하는 과정에서 요청값과 응답값이 맞지 않아 오류가 반복되는 문제가 있었습니다.

### 해결 방법

기능 구현에만 집중하기보다, 팀원들이 같은 기준으로 개발할 수 있도록 기능별 API 명세를 다시 정리했습니다.  
단어 학습, 글 요약 학습, 커뮤니티 기능별로 요청 파라미터, 응답 예시, 오류 상황을 Notion에 공유하며 연동 기준을 맞췄습니다.

### 배운 점

코드 구현뿐 아니라 명확한 문서화와 커뮤니케이션이 협업 과정의 문제를 줄이는 데 중요하다는 것을 배웠습니다.

## 실행 방법

> 실행 환경과 설정 파일은 로컬 개발 환경에 맞게 수정해야 합니다.

### 1. Repository clone

```bash
git clone https://github.com/JEON-YongMin/SlowLearning.git
cd SlowLearning/BackEnd
```

### 2. DB 설정

`src/main/resources` 경로의 설정 파일에서 로컬 DB 정보를 설정합니다.

예시:

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/slowlearning
    username: your_username
    password: your_password
```

민감한 정보가 포함된 설정 파일은 GitHub에 업로드하지 않고 로컬에서만 관리합니다.

### 3. 프로젝트 실행

Mac / Linux:

```bash
./gradlew bootRun
```

Windows:

```bash
gradlew.bat bootRun
```
